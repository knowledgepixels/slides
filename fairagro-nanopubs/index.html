🎯 Takeaways

  • Nanopublications are small FAIR data packages
  • They can describe anything: data points, datasets, studies, messages, opinions, reviews, social links, etc.
  • They are the basis of an ecosystem that provides a globally integrated open knowledge graph
  • They are still rapidly evolving but ready to be used

What are nanopublications?

  • Smallest unit of attributable, structured knowledge
  • Three graphs: assertion, provenance, publication info
  • RDF, content-addressed, digitally signed
  • nanopub.net

✨ How nanopublications are FAIR

  • Findable — globally unique, persistent identifiers (Trusty URIs)
  • Accessible — retrievable from an open, decentralized network
  • Interoperable — self-describing RDF using shared vocabularies
  • Reusable — provenance, license, and signature travel with each claim

🧩 Use cases

  • FAIR Digital Objects — a building block and unifying layer for FDOs
  • Vocabularies & ontologies — define and publish terms on the fly
  • Biodiversity domain modeling — e.g. taxon–environment relations
  • News items — structured, citable, machine-readable updates
  • Personal research profiles — talks attended, papers read, and more
  • Corrections & suggestions — feedback on existing work
  • FAIR implementation decisions — record choices and their rationale
  • … and much more

🗄️ Reliable publishing

Nanopub Registry

  • Validates and stores published nanopublications
  • Copied across independent servers, so nothing gets lost
  • Records stay put — retracted when needed, never deleted

🔍 Global querying

Nanopub Query

  • Makes all nanopublications searchable and discoverable
  • Organized by author and topic, with ready-made search templates
  • A separate layer, so searching never slows down publishing

🔌 Nanopub Elements

  • Ready-made web components to show nanopublications on any website
  • Display them as lists, tables, or single items
  • Just add an HTML tag — no coding required
  • github.com/nanopublication/nanopub-elements
